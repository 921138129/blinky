blinky

a fisheye experiment for wide FOV gameplay

abstract

background Fisheye Quake was created by Aardappel in 1999. It allows you to play Quake with a very wide Field of View. It does this by rendering six standard 90° views to a cubemap, thereby capturing all surrounding pixels like a Skybox. All pixels are then mapped to the screen using a fisheye projection.

results Blinky is a new mod that extends Fisheye Quake by allowing user-customizable views in the form of Lua scripts. We do this by introducing the concept of "Globes and Lenses", formally known as environment maps and projections, respectively. A Globe script defines how to capture the surrounding pixels (like a cubemap), and a Lens script defines how to project those pixels to the screen (like a fisheye projection). By using this generic interface, we are able to experiment with bizarre new views. Many Lens scripts are created from existing map projections from cartography and panoramic photography tools.

conclusions We find that a tetrahedron Globe renders faster than a cubemap but at the cost of picture quality. We also find that stereographic and Panini Lenses seem the most practical in real gameplay in comparison with other wide-angle Lenses that seem more nauseating.

install

Windows

Windows users must extract the folder to their Quake directory and run tyr-quake.exe to start.

Linux

Linux users will have to compile from source. LuaJIT is a required library.

After installing LuaJIT, run the following to build: 

$ make prepare
$ make tyr-quake
Now copy tyr-quake to your quake directory, and copy the globes/ and lenses/ folders to your $HOME/.tyrquake directory.

Commands

Bring down the console with the tilde (~) key and you can type the following commands: 
globe <name> : choose a globe (affects picture quality and render speed)
lens <name> : choose a lens (affects the shape of your view)
hfov <degrees> : zoom by specifying horizontal FOV
vfov <degrees> : zoom by specifying vertical FOV
hfit : zoom by fitting your view horizontal bounds in the screen
hfit : zoom by fitting your view's vertical bounds in the screen
fit : zoom by fitting your whole view in the screen
rubix : display colored grid for each rendered view in the globe
Note: the globe and lens commands both have tab completion

Examples (as seen in video above)

A very usable modern panoramic projection called Panini: 

globe cube
lens panini
hfov 180

A full map projection used by National Geographic: 

globe tetra
lens winkeltripel
fit
